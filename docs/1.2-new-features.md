# New Features in Pramnos Framework v1.2

## Database

### Read/Write Replicas Support
The `Database` class now supports separate connections for read and write operations. This allows scaling the database by using read replicas for SELECT queries.

#### Configuration
In your `settings.php`, you can now define `read` and `write` configurations:

```php
'database' => [
    'type' => 'mysql',
    'read' => [
        'hostname' => 'db-read.example.com',
        'user' => 'readonly',
        'password' => 'secret',
        'database' => 'myapp'
    ],
    'write' => [
        'hostname' => 'db-write.example.com',
        'user' => 'root',
        'password' => 'secret',
        'database' => 'myapp'
    ],
    'port' => 3306,
    'prefix' => 'pramnos_',
    'collation' => 'utf8mb4_unicode_ci'
]
```

#### Usage
The framework automatically detects if a query is a "write" query (INSERT, UPDATE, DELETE, etc.) or a "read" query (SELECT, SHOW, EXPLAIN) and uses the appropriate connection.

```php
$db = Database::getInstance();

// Uses READ connection automatically
$users = $db->query("SELECT * FROM #PREFIX#users");

// Uses WRITE connection automatically
$db->query("UPDATE #PREFIX#users SET last_login = NOW() WHERE userid = 1");
```

### Connection Health & Auto-reconnect
The framework now automatically detects if a database connection has been lost (e.g., during long-running background tasks) and attempts to reconnect once before failing a query.

This is transparent to the developer and helps improve the stability of background workers and daemons.
